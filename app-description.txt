PROJECT OVERVIEW
================================================================================
A locally-hosted web application (Node.js + vanilla JavaScript) for managing 
child admission sessions at an amusement center. All data persists server-side 
with no internet connection required. Interface fully in Azerbaijani language.
Features real-time session tracking, analytics, revenue reporting, and daily data export.

IMPLEMENTATION NOTES & RATIONALE
================================================================================

Timer Efficiency: Updates Every 60 Seconds (Not Every Second)
  Rationale: Significantly reduces DOM updates and server load for long-running sessions
            Color changes are responsive at minute-level granularity (sufficient UX)
            Prevents unnecessary re-renders and CPU usage spikes

Pastel Colors + Dark Borders vs. Solid Colors
  Rationale: Better visual definition and professional appearance
            Ensures readable text contrast throughout interface
            Dark borders distinguish buttons from background more effectively
            Consistent with modern UI design patterns

Full Hours Only for Revenue Calculation
  Rationale: Simplifies billing logic and reduces customer disputes
            Standard practice in service industries
            Clear, predictable pricing structure
            Reduces complex partial-hour rounding issues

Unlimited Sessions Display "Limitsiz" Instead of Symbol
  Rationale: Azerbaijani text more intuitive than infinity symbol (∞) for local users
            Maintains UI language consistency throughout app
            Cleaner table appearance without special character encoding issues

Confirmation Dialogs for Session Changes
  Rationale: Prevents accidental session modifications (End, Extend, Delete)
            Provides user confidence in long-running sessions
            Standard UX pattern for critical operations
